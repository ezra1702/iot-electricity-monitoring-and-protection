# Cara Menjalankan Project VoltEdge Energy Dashboard dengan Docker

Project ini menggunakan Docker Compose untuk mempermudah proses instalasi dan menjalankan semua layanan sekaligus (Frontend, Backend, Database MySQL, MQTT, dan phpMyAdmin).

## Langkah-langkah Menjalankan:

1. Buka terminal atau command prompt (CMD/PowerShell) di dalam folder project ini.
2. Jalankan perintah berikut:

   docker-compose up -d --build

   Penjelasan perintah:
   - "up" : Digunakan untuk membuat dan menjalankan container.
   - "-d" : (Detached mode) Menjalankan container di background/latar belakang, sehingga kamu masih bisa menggunakan terminal.
   - "--build" : Memastikan Docker mem-build ulang image dari Dockerfile jika ada perubahan kode terbaru.

   ⚠️ JIKA ADA PERUBAHAN FILE TAPI TIDAK TERBACA OLEH DOCKER (KARENA CACHE):
   Gunakan perintah ini untuk force-rebuild secara bersih tanpa cache:
   docker-compose build --no-cache && docker-compose up -d

## Daftar Akses Layanan (URL / Port):

Setelah perintah di atas selesai dijalankan, kamu bisa mengakses layanan-layanan berikut di browser:

- Frontend (Website Utama) : http://localhost:3000
- Backend (API)            : http://localhost:5000
- phpMyAdmin (GUI Database): http://localhost:8081 
  (Login dengan Username: root & Password: rootpassword)
  *Catatan: Port menggunakan 8081 agar tidak bentrok dengan XAMPP atau Docker phpMyAdmin dari project lain yang mungkin menyala di 8080.*
- MQTT Broker              : Berjalan di port 1883 dan 9001 (WebSocket)
- MySQL Database           : Berjalan di port 3306 (internal Docker)

## 🗄️ Dimana Database Disimpan?
Berbeda dengan XAMPP yang menyimpan database di folder lokal komputer (misal C:\xampp\mysql\data), Docker menyimpan database ini di dalam **Docker Volume** (terisolasi dan aman). 
Data ini tidak akan hilang walau container dimatikan, KECUALI kamu menjalankan perintah penghapusan volume. Semua manajemen isi database bisa kamu lakukan secara visual melalui `http://localhost:8081`.

## Perintah Penting Lainnya:

- Untuk menghentikan semua layanan (tanpa menghapus data):
  docker-compose stop

- Untuk menghentikan DAN menghapus container (data di volume tetap aman):
  docker-compose down

- ⚠️ Untuk MENGHAPUS SEMUA TERMASUK DATABASE (Hati-hati!):
  docker-compose down --volumes

- Untuk melihat log error atau proses dari semua layanan secara real-time:
  docker-compose logs -f

- Jika ingin melihat log layanan tertentu (misal hanya backend):
  docker-compose logs -f backend
